## Контекст приложения

- Telegram Mini App для покупки игрового доната (redonate.su)
- Пользователи — геймеры, аудитория 14-35 лет
- Тёмная тема по умолчанию (как в Telegram)
- Используй Telegram CSS variables: `--tg-theme-bg-color`, `--tg-theme-text-color` и т.д.

## Принципы дизайна

- Минимализм: чисто, без лишнего шума
- Скорость: анимации 200-300ms, не тормозит
- Мобайл-фёрст: экраны телефонов, свайпы, bottom sheet'ы
- Геймерская эстетика: акценты неоновых цветов (синий/фиолетовый), градиенты, свечение
- Доступность: контраст минимум 4.5:1, touch targets минимум 44px

## Правила кода

- Только функциональные компоненты + хуки
- Tailwind утилиты, не inline styles
- Анимации через Framer Motion, не CSS keyframes
- Компоненты в `miniapp/src/components/`
- Не использовать устаревшие паттерны (классовые компоненты, Redux без необходимости)

## Диагностический флоу

При баге — трейсируй полный путь до нахождения owner layer:
```
route → layout → page → hook → API call → backend → DB → response → UI state
```

**Правило:** никогда не добавляй fallback-рендеринг чтобы скрыть отсутствующие или неверные данные — исправляй источник данных.

Если данные от API неверные — это баг backend'а. Сообщи teamlead, не патчи на стороне UI.

При изменении экрана — обновить все связанные части:
- [ ] Компонент
- [ ] Хук, который получает данные
- [ ] Loading и error состояния
- [ ] Типы (если изменился shape данных от API)

## Стоп-условия — сообщи teamlead

- Root cause бага в backend или API контракте
- Нужна новая архитектура роутинга
- Изменения затрагивают API контракт
